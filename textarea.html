Textarea

The textarea component uses semantic HTML and BEM-structured classes to create floating labels and validation feedback. Every textarea begins with the base .textarea class, while supporting elements like .textarea__label and .textarea__error provide structure for labels and error messages.

  • .textarea — base class for styling, spacing, and behavior
  • .textarea__group — wrapper that positions label and textarea
  • .textarea__label — floating label that animates on focus or when text is entered
  • .textarea__error — validation message shown on error

Textareas are wrapped inside .textarea__group for layout consistency. The label floats above the textarea when it is focused or contains text, and error messages appear beneath the textarea using .textarea__error.

Example markup:


<div class="textarea__group">
  <textarea
    class="textarea"
    id="observations"
    name="observations"
    rows="4"
    placeholder=" "
    required
  ></textarea>
  <label for="observations" class="textarea__label">Observations</label>
  <div class="textarea__error">This field is required.</div>
</div>

  background: transparent;

  /* --- Floating label --- */
  /* Slide label up on focus OR when textarea has content */
  /* --- Group wrapper --- */
}


.textarea::-moz-placeholder {
  color: transparent;
}


.textarea::placeholder {
  color: transparent;
}


.textarea:required, .textarea:invalid {
  box-shadow: none;
}


.textarea:focus {
  background-color: var(--clr-light) !important;
  border-color: #087EA4;
}


.textarea:-moz-placeholder ~ .textarea__label {
  transform: translateY(0%) scale(1);
}


.textarea:placeholder-shown ~ .textarea__label {
  transform: translateY(0%) scale(1);
}


.textarea:not(:-moz-placeholder) ~ .textarea__label {
  transform: translateY(-100%) scale(0.9);
  color: #087EA4;
}


.textarea:focus ~ .textarea__label, .textarea:not(:placeholder-shown) ~ .textarea__label {
  transform: translateY(-100%) scale(0.9);
  color: #087EA4;
}
