Form Structure & Submission

The Form Component provides a flexible, accessible structure for collecting user input through various input types (text fields, radio buttons, file uploads, etc.) and submitting data through styled action buttons. It also integrates a built-in Loader element for visual feedback during submission.

This component follows semantic HTML form conventions while offering a consistent design system across inputs and actions. The loader is contained inside the submit button to indicate background processing or pending actions.

  • .form__container — the outer wrapper that defines the overall layout and spacing of the form.
  • .form — the main form element used to group form fields, labels, and actions.
  • .form__actions — container for all actionable buttons such as submit, cancel, or reset.
  • .button — base button styling applied to form actions.
  • .button--primary — modifier indicating a main action button (e.g., form submission).
  • .loader — animated loader that visually represents ongoing processing.
  • .loader__circle — individual animated element inside the loader that forms the spinning visual.
  • .xs — optional size modifier for the loader to keep it small and unobtrusive within buttons.

Accessibility Notes:

  • Always include appropriate label elements for input fields to maintain accessible form design.
  • The loader is purely decorative and should use aria-hidden="true" if possible to avoid distraction for screen readers.
  • Buttons should include descriptive type attributes (e.g., type="submit") for proper form behavior.

Usage Tips:

  • Use .form__container to maintain spacing and alignment of all form elements.
  • Place loaders inside action buttons to indicate pending state during submissions.
  • Keep form layouts consistent across pages for a cohesive user experience.

Customization Notes:

  • The .loader can be resized using modifiers such as .xs, .sm, or .lg depending on available space.
  • Combine .button--primary with color variants to align with your theme.

Example markup:



  <div class="form__container">
    <form class="form" method="POST">

      <p>Add inputs, radio, fileUpload, etc. here</p>

      <div class="form__actions">
        <button class="button button--primary" type="submit">
          Save
          <div class="loader xs" aria-hidden="true">
            <div class="loader__circle"></div>
            <div class="loader__circle"></div>
          </div>
        </button>
      </div>

    </form>
  </div>

this Tiny helper that, given a form, walks its inputs in order and focuses the first one that’s visible (skips display:none or hidden). If the next sibling is a <label tabindex="0" > it picks that instead. Does nothing for invalid/non-form nodes.

Detailed instructions for integrating JavaScript functionality are available here.

JS:




// Form Validation

const isValidFormElement = (form) => {

  return form && form.tagName && form.tagName.toLowerCase() === 'form';

};


// Input Focus Utilities

const isFocusableLabel = (element) => {

  return element?.matches('label[tabindex="0"]');

};

const isInputVisible = (input) => {

  return !input.matches('[style*="display: none"]') && !input.hasAttribute('hidden');

};

const focusElement = (element) => {

  element.focus();
};

const findFirstFocusableElement = (form) => {

  let currentElement = form.querySelector('input')
  ;
  
  while (currentElement) {
    if (isFocusableLabel(currentElement.nextElementSibling)) {
      return currentElement.nextElementSibling;

    } else if (isInputVisible(currentElement)) {

      return currentElement;

    } else {

      currentElement = currentElement.nextElementSibling;

    }

  }
  
  return null;
};

// Main Focus Function
export const focusNonHiddenInput = (form) => {

  if (!isValidFormElement(form)) {

    return;

  }

  const focusableElement = findFirstFocusableElement(form);

  
  if (focusableElement) {

    focusElement(focusableElement);

  }
  
};
