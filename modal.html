Modal

Modal Component

The Modal component provides a centered overlay dialog for displaying content such as messages, forms, or tables without leaving the current page. It remains hidden by default and becomes visible when the .open class is added.

It includes a flexible structure with a header, a scrollable body area, and an optional sticky footer for action buttons. The .transparent modifier hides the header and background for minimalist use cases, such as image previews or embedded content.

  • .modal — the full-screen overlay with a translucent, blurred backdrop.
  • .modal.open — visible and interactive state.
  • .modal__content — the inner dialog container managing layout, animations, and transitions.
  • .modal__header — top section with a title and optional controls.
  • .modal__closer — circular “X” button positioned slightly outside the top-right corner.
  • .modal__body — main content area with flexible vertical layout.
  • .modal__body__content — scrollable inner wrapper for text, images, or forms.
  • .form__actions — optional footer for buttons, fixed at the bottom inside the modal.
  • .transparent — variant modifier hiding the header and removing shadows/background color for simple content.

The modal fades and slides into view using transitions on opacity and transform, with flexible centering via flexbox.

This CSS sheet controls the overlay and the dialog box that pops up when the user clicks the open button. It keeps everything hidden until JavaScript adds the class open; then the overlay fades in, the dialog slides down, and the rest of the page is blurred.

  • .modal is a full-screen layer that starts invisible (opacity: 0 and z-index: -1). When .open is added it jumps to z-index: 15, fades to full opacity, and lets clicks reach the dialog (pointer-events: auto).
  • .modal__content is the actual white box: rounded corners, shadow, and a slide-in animation (transform: translateY(-3rem) → 0).
  • .modal__header holds the title and the round close button; it can be hidden with the extra class transparent (useful for image-only pop-ups).
  • The close button (.modal__closer) sits half-outside the corner, scales on hover, and gets a visible focus ring for keyboard users.
  • .modal__body__content scrolls if the content is tall, while .form__actions stays glued to the bottom so buttons are always reachable.
  • Everything is capped at 90 % of the viewport width/height, so the modal never overflows the screen.

JS 


// Import utility to handle 'inert' state management
import { toggleInertForAllExceptOpenedElement } from "../utils/Inert";


// Import utility to set appropriate ARIA attributes for accessibility
import { setAriaAttributes } from "../utils/Aria";


// ======================
// DOM ELEMENT SELECTORS
// ======================

const getModalElements = () => ({

  openButton: document.querySelector(".modal__opener"),
  closeButton: document.querySelector(".modal__closer"),
  modal: document.querySelector(".modal"),

});

const hasValidModalElements = ({ modal, openButton, closeButton }) => {

  return !!(modal && openButton && closeButton);

};

// ======================
// MODAL STATE MANAGEMENT
// ======================

const getModalState = (modal) => {

  return modal.classList.contains("open");

};

const setModalState = (modal, isOpen) => {

  if (isOpen) {

    modal.classList.add("open");
    const modalCloser = modal.querySelector(".modal__closer");
    modalCloser.focus();

  } else {

    modal.classList.remove("open");

  }

};

// ======================
// ACCESSIBILITY MANAGEMENT
// ======================
const updateModalAccessibility = (modal, isOpen) => {

  const tabIndex = isOpen ? "0" : "-1";
  setAriaAttributes(!isOpen, tabIndex, modal);

};

const manageFocus = (closeButton, shouldFocus) => {

  if (shouldFocus) {
    closeButton.focus();
  }

};

// ======================
// INERT STATE MANAGEMENT
// ======================
const updateInertState = (modal, isOpen) => {

  toggleInertForAllExceptOpenedElement(modal, "open");

};

// ======================
// MODAL TOGGLE LOGIC
// ======================
const toggleModal = (modal, closeButton) => {

  const wasOpen = getModalState(modal);
  const willBeOpen = !wasOpen;

  // Update visual state
  setModalState(modal, willBeOpen);

  // Update accessibility
  updateModalAccessibility(modal, willBeOpen);

  // Update inert states
  updateInertState(modal, willBeOpen);

  // Manage focus
  manageFocus(closeButton, !willBeOpen);

};

// ======================
// EVENT HANDLERS
// ======================
const createOpenHandler = (modal, closeButton) => () => {

  toggleModal(modal, closeButton);

};

const createCloseHandler = (modal, closeButton) => () => {

  toggleModal(modal, closeButton);

};

// ======================
// EVENT BINDING
// ======================
const bindModalEvents = (openButton, closeButton, modal) => {

  const openHandler = createOpenHandler(modal, closeButton);
  const closeHandler = createCloseHandler(modal, closeButton);

  openButton.addEventListener("click", openHandler);
  closeButton.addEventListener("click", closeHandler);

};

// ======================
// MAIN INITIALIZER
// ======================
const Modal = () => {

  const elements = getModalElements();

  if (!hasValidModalElements(elements)) {
    console.warn("Modal: Required elements not found");
    return;

  }

  const { openButton, closeButton, modal } = elements;
  bindModalEvents(openButton, closeButton, modal);

};

export default Modal;

This JavaScript module wires up the modal . It shows the box when the user clicks the “open” button, hides it when the “close” button (the ×) is clicked, and keeps the experience friendly for keyboard users and screen-reader users.

  • toggleInertForAllExceptOpenedElement makes everything except the modal unfocusable while the dialog is open, so tabbing can’t accidentally leave the box.
  • setAriaAttributes flips aria-hidden and tabindex so assistive technology knows when the modal is visible or hidden.
  • When the modal opens, keyboard focus is moved immediately to the × button; when it closes, focus is returned to the element that triggered it.
  • All state changes (open/close) are done by simply adding or removing the CSS class open; the rest (fade-in, slide-down, blur, etc.) is handled by the CSS we wrote earlier.
  • The module self-initialises: import it once (import Modal from './modal'; Modal();) and it finds the correct buttons and modal element automatically.
  • For more information about JavaScript integration, visit the documentation [here].
