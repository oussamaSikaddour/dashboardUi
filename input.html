Form Inputs

The input component uses semantic HTML and BEM-structured utility classes to create floating labels, icons, and validation feedback without relying on external libraries. Every input begins with the base .input class, while supporting elements like .input__label, .input__span, and .input__error provide structure for icons and messages.

  • .input — base class for styling, spacing, and behavior
  • .input__group — wrapper that positions label and icons
  • .input__label — floating label that animates on focus or input
  • .input__span — optional icon (e.g., visibility toggle or dropdown)
  • .input__error — validation message shown on error

Inputs are grouped inside .input__group to align floating labels and icons. Validation messages appear beneath the input using .input__error, and icons inside .input__span can trigger interactions, such as toggling password visibility via JavaScript.

Example markup:


<div class="input__item">
  <div class="input__group">
    <input class="input" type="text" placeholder=" " required />
    <label class="input__label">Username</label>
    <span class="input__span">
      <i class="fa-solid fa-user"></i>
    </span>
  </div>
  <p class="input__error">This field is required.</p>
</div>
